# Женско-мужской переводчик (мультиязычный)

Сервер-шутка: переводит «что сказано» ↔ «что имелось в виду» на **10 языках**
(en, ru, lv, ru-latn, lt, et, pl, de, uk, es).

## Как работает перевод (концепт-хранилище)

Один **концепт** = одна единица смысла (фраза + расшифровка). Внутри концепта —
варианты на всех языках (`variants: {lang:{phrase,meaning}}`) и **один эмбеддинг
каноничной английской фразы** (пивот). Поэтому фраза на любом языке попадает в
тот же концепт, и найденное на одном языке сразу доступно на других.

```
фраза P (язык L)
  └─▶ 1. пивот: P → английский (если L≠en)
        └─▶ 2. embed(en) → косинус по концептам (все эмбеддинги английские)
              ├─ хит + есть L     → отдаём из кэша
              ├─ хит, нет L       → доматериализуем L (Gemini) в тот же концепт
              └─ промах           → decode смысла (en) + локализация в L, новый концепт
```

Порог склейки — `SIMILARITY_THRESHOLD` в `.env` (по умолчанию 0.82).

## Тон перевода (важно!)

Тот, кто пользуется этим переводчиком, **любит** человека противоположного пола,
которому переводит. Поэтому расшифровки делаются **смешно и легко (casual)** на первом
плане, с тёплым фоном. Но при этом — **не сглаживаем углы**: настоящий смысл передаём
**точно и честно**, даже если он чуть колкий; не смягчаем и не разбавляем. **Без сюсюканья
и приторности**, и **никогда** цинично, с угрозой, сарказмом или издёвкой.

Где это закодировано:
- `src/GoogleAi.php` → `decode()` — промпт генерации смысла (warm + loving + casual + funny);
- `src/GoogleAi.php` → `localize()` / `localizeMany()` — сохраняют тёплый-смешной тон при переводе на другие языки;
- `src/Translator.php` → словари `SHE`/`HE` — курируемые фразы уже написаны в этом тоне (эталон).

Меняешь тон — правь промпты в `decode()`/`localize*()` и эталонные смыслы в словаре, затем пересобери базу: `rm data/translator.sqlite* && php bin/fill.php`.

## Структура

```
translator/
├─ public/
│  ├─ index.html            — фронтенд (мобильный; выбор языка, на домашний экран)
│  └─ api.php               — JSON API
├─ src/
│  ├─ Lang.php              — реестр 10 языков (код, метка, имя для промптов)
│  ├─ GoogleAi.php          — клиент Gemini: embed / toEnglish / decode / localize
│  ├─ Db.php                — SQLite: концепты (варианты всех языков + эмбеддинг)
│  ├─ ConceptService.php    — оркестрация пивот→кэш→доматериализация + косинус
│  ├─ Translator.php        — стартовый русский словарь (источник для прогрева)
│  └─ Env.php               — загрузчик .env
├─ bin/seed.php             — прогрев: словарь → концепты (ru→en пивот)
├─ bin/fill.php             — полное заполнение: сид (если пусто) + ВСЕ 10 языков у всех концептов
├─ data/translator.sqlite   — база (создаётся сама; в .gitignore)
├─ router.php               — роутер встроенного PHP-сервера
├─ .env(.example)           — ключ Google AI + порог похожести
└─ README.md
```

## Настройка Google AI

1. Получи ключ: <https://aistudio.google.com/apikey>
2. Впиши в `.env`: `GOOGLE_AI_API_KEY=...` (перезапуск не нужен).
3. (Опц.) прогрей словарь в векторный кэш: `php bin/seed.php`

### Заполнить все языки сразу

```bash
php bin/fill.php   # сидит словарь (если база пуста) и доливает все 10 языков в каждый концепт
```

`fillAll()` для каждого концепта одним запросом к Gemini переводит фразу+смысл на все
недостающие языки и кладёт их в ту же запись (`localizeMany`). Идемпотентно: уже
заполненные языки пропускаются.

Модели: эмбеддинги `gemini-embedding-001` (768d), генерация `gemini-flash-lite-latest`.
Без ключа всё работает в режиме «только словарь».

> Заметка: на текущем ключе `gemini-2.0-flash` отдаёт `quota limit: 0`
> (нет free-tier на генерацию), поэтому используется `gemini-flash-lite-latest`.
> Список доступных моделей: `GET /v1beta/models?key=...`.

## Запуск

```bash
cd /var/www/translator
php -S 0.0.0.0:8088 -t public router.php
```

Открыть: <http://localhost:8088>  (с телефона в той же сети — `http://<ip-компа>:8088`).

## API

```
POST /api/translate  {"text":"я не злюсь","direction":"f2m","lang":"ru","mood_she":50,"mood_he":50}
   → {"text":"<смысл на lang>","source":"ai","direction":"f2m","lang":"ru"}
     source: cache | cache-new-lang | ai | ai-mood | no-key   (cache* добавляет "similarity")
     mood_she / mood_he: 0 (😊 happy) … 100 (😠 angry), середина=нейтрально.
       сдвиг любого >10 от центра → расшифровка под настроение (source "ai-mood", без кэша);
       настроение говорящего = «как сказала/сказал», слушающего = «как услышал».
GET  /api/langs   → {"langs":[{code,label,flag}...], "ai_enabled":true}
GET  /api/stats   → {"concepts":..,"ai":..,"dict":..,"hits":..,"variants_lang":{lang:count}}
```

`direction`: `f2m` (женский→мужской) · `m2f` (мужской→женский).
`lang`: en · ru · lv · ru-latn · lt · et · pl · de · uk · es.

## Добавить фразы

Правится один файл — `src/Translator.php`, массивы `SHE` / `HE`:

```php
['новая фраза', 'её настоящий смысл'],
```

Сервер перечитывает файл на каждом запросе — перезапуск не нужен.
